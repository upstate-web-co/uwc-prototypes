# uwc-prototypes

Admin-owned, UWC-internal repo. Every portal-originated client project's design-phase prototype lives on a branch here (`prototype/<project_id>`), gets generated by a headless Claude Code run dispatched from agency-admin, and deploys to a CF Pages preview subdomain with HTTP basic auth + `noindex`.

Rule 72. See [uwc-agency-admin/CLAUDE.md](https://github.com/upstate-web-co/uwc-agency-admin/blob/main/CLAUDE.md) for the authoritative contract.

## How it works

1. Portal client approves questionnaire at discovery → admin's `phase-complete.ts` fires `generate-prototype(projectId)`.
2. Admin dispatches `uwc-prototype-build.yml` on this repo with `project_id + prompt + job_id`.
3. Workflow checks out / creates `prototype/<project_id>`, runs Claude Code (30-turn max), commits changes, builds Astro, deploys to CF Pages under `prototype-<project_id>`, screenshots the live URL, and POSTs results back to admin via HMAC webhook.
4. Admin stamps `projects.prototype_url` + `prototype_generated_at` and notifies the client via the portal.

## Required secrets

Repo → Settings → Secrets and variables → Actions:

- `ANTHROPIC_API_KEY` — Claude Code's token for the 30-turn run
- `CLOUDFLARE_API_TOKEN` — CF Pages deploy token (Pages:Edit scope)
- `CLOUDFLARE_ACCOUNT_ID` — UWC's CF account id
- `PORTAL_TO_ADMIN_HMAC_SECRET` — shared with admin for the return webhook

## Required installs

- Claude Code GitHub App must be installed on this repo (same install used across `upstate-web-co/*`)

## Not a template

This isn't a template repo per se — it's a single repo with per-project branches. Branches live for the lifetime of the project; after prototype approval (Rule 73), the actual build happens on the client's own repo (`upstate-web-co/<client-slug>`).

## What's on `main`

A minimal Astro + Tailwind scaffold forked from [`uwc-base-template`](https://github.com/upstate-web-co/uwc-base-template). Each prototype branch iterates from this base.
